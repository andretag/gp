11 ## Structure
12 All source code is located in `src/`; headers are in `include/`; unit tests are in `/test/`; and executables are in `exec/`. Compiled binaries will be placed in `bin/`.

14 ## Dependencies
15 I may miss a few here, but here is a list of dependencies:
16 
17 * [Eigen](http://eigen.tuxfamily.org/dox/) (header-only linear algebra library)
18 * Gflags (Google's command-line flag manager)
19 * Glog (Google's logging tool)
20 
21 All of these may be installed very easily. If you run into any trouble, though, I am more than happy to help you figure out what's going on. Just post an [issue](https://github.com/dfridovi/gp/issues) on this repository and I will reply as soon as possible.

23 ## Usage
24 You'll need to begin by building the repository. From the top directory, type the following sequence of commands:
25 
26 ```
27 mkdir bin
28 mkdir build
29 cd build
30 cmake ..
31 make -j4
32 ```
33 
34 This should build all tests and executables. In order to run tests, you can run the following command:
35 
36 ```
37 ./run_tests
38 ```
39 
40 from within the `build/` directory you just made. All the tests should pass, and none should take more than a second or so to run.
41 
42 Executables are automatically placed within the `bin/` directory that you created. To run them, just type `./(name-of-executable)`.
43 
44 To the extent that it makes sense, all parameters are accessible from the command line via Gflags. For help with command line options, simply run the following command:
45 
46 ```
47 ./(name-of-executable) --help
48 ```
